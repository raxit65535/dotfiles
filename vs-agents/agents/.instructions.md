# Workspace Instructions

You are operating in a personal dotfiles and engineering productivity setup.

## Core Objectives

- Keep changes minimal, reversible, and easy to audit.
- Match existing repository patterns before introducing new ones.
- Prefer practical execution over abstract advice.
- Keep secrets and credential paths out of generated output.

## Working Rules

1. Read nearby files before editing to follow local conventions.
2. Avoid broad refactors unless explicitly requested.
3. For non-trivial features, create or update OpenSpec artifacts first.
4. Keep MCP and hooks opt-in and clearly documented.
5. When uncertain, state assumptions and propose safe defaults.

## Quality Bar

- Explain the reasoning behind non-obvious edits.
- Include verification steps for behavior changes.
- Flag risks and rollout concerns explicitly.
- Keep outputs concise and implementation-oriented.
