# Agents Catalog

These agents are intentionally aligned with your existing `~/.agents` style so
behavior feels familiar in Copilot Chat.

## Core Delivery

- `dev-backend`: backend implementation
- `dev-frontend`: frontend implementation
- `test-backend`: backend test authoring and hardening
- `test-frontend`: frontend/UI test authoring and hardening
- `spec-writer`: OpenSpec planning for non-trivial changes
- `gatekeeper`: quality gate and readiness verdict
- `principal-engineer`: architecture and risk review

## Platform and Support

- `drone-ci`: CI pipeline changes and troubleshooting
- `infraspec`: infrastructure spec edits and review
- `designer`: design and UX-to-implementation handoff

## Usage Tips

1. Start with `spec-writer` for non-trivial work.
2. Use implementation agents for coding.
3. Use testing agents to close coverage gaps.
4. End with `gatekeeper` or `principal-engineer` review.
