# Project Plan Skill

Use this skill to convert a broad feature idea into a practical plan that can be
executed in phases.

## Workflow

1. Clarify problem statement and target users.
2. Capture goals, non-goals, constraints, and dependencies.
3. Define success criteria and acceptance checks.
4. Produce phased task breakdown with dependencies.
5. Call out open questions and risk mitigations.

## Output Template

- Summary
- Problem and motivation
- Proposed approach
- Scope (in/out)
- Task phases
- Validation and rollout
- Open questions
