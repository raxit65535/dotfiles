# Skills Catalog

Each skill is packaged in its own folder with a `SKILL.md` file. This is why
subfolders exist under `skills/`.

Example:

- `skills/openspec-workflow/SKILL.md`

## Included Skills

- `openspec-workflow`: planning and proposal structure
- `project-plan`: converting rough ideas into implementation-ready plans
- `openspec-execute`: phased task execution guidance
- `validate-changes`: validation checklist before completion
- `awesome-copilot-adoption`: safe external content adoption process
- `repo-conventions`: local repository conventions and safety rules

## How to Invoke

In Copilot Chat, describe intent directly, for example:

- "Plan this feature using OpenSpec workflow"
- "Execute this OpenSpec tasks list in phases"
- "Validate my change before I mark it done"
