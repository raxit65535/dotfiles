# Neovim setup

This Neovim config is intentionally optimized for **reading and reviewing**, not for full-time coding.

## Primary workflows

1. **Markdown review** for planning files generated by Copilot or other agents
2. **Git change review** similar to a lightweight source-control workflow
3. **Light terminal access** when needed, without a dedicated terminal plugin
4. **Review-first startup dashboard** when opening Neovim with no file target

## Layout

- `init.lua` — tiny entrypoint
- `lua/config/options.lua` — editor defaults
- `lua/config/keymaps.lua` — global keymaps
- `lua/config/autocmds.lua` — filetype and terminal behavior
- `lua/config/lazy.lua` — plugin bootstrap
- `lua/plugins/ui.lua` — statusline, keymap discovery, startup dashboard
- `lua/plugins/navigation.lua` — file browsing and fuzzy finding
- `lua/plugins/git.lua` — hunk review, diff view, git status workflow
- `lua/plugins/markdown.lua` — Treesitter + markdown rendering

## Why these choices

- **nvim-tree** for a familiar sidebar explorer that behaves closer to VS Code
- **Telescope** for quickly opening generated markdown, tracked files, and changed files
- **Gitsigns + Diffview + Neogit** for working-tree diffs, file history, and repo history in a side-by-side review flow
- **Render Markdown** for better readability of planning docs
- **Built-in `habamax` theme** from Neovim itself instead of a custom colorscheme plugin
- **Built-in terminal** instead of ToggleTerm to keep the setup lean
- **Mini Starter** for a lightweight startup dashboard without another heavy dependency

## Keymaps

### Navigation

- `<leader>e` — toggle explorer sidebar
- `<leader>E` — focus explorer sidebar
- `-` — reveal current file in explorer
- `<C-p>` / `<leader>ff` — find files
- `<leader>fg` — grep
- `<leader>fb` — buffers
- `<leader>fr` — recent files
- `<leader>fc` — changed files
- `<leader>gf` — git-tracked files
- `<leader>st` — open start screen

### Git review

- `<leader>gg` — open git status in a full tab
- `<leader>do` — open working-tree diff side by side
- `<leader>dc` — close diff view
- `<leader>df` — toggle changed-files panel
- `<leader>dv` — focus changed-files panel
- `<leader>dh` — current file history
- `<leader>dH` — repo commit history
- `]h` / `[h` — next/previous hunk
- `<leader>gp` — preview hunk
- `<leader>gs` — stage hunk
- `<leader>gr` — reset hunk

### Markdown

- `<leader>mr` — toggle rendered markdown view

### Startup dashboard

When Neovim opens without a file target, it lands on a lightweight dashboard with actions for:

- working-tree diff
- repo commit history
- git status
- markdown plan search
- recent files in the current directory

### Terminal

- `<leader>tt` — open a horizontal terminal split
- `<Esc><Esc>` — leave terminal mode
